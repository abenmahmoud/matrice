Génération IA — fonctionnement

Modèle :  GPT-5.4 via proxy Replit AI Integrations (→ gpt-4o sur VPS)
Format :  response_format: json_object — toutes les réponses sont du JSON structuré parsé et validé
Fallback :  si l'API IA échoue, chaque module retourne un contenu déterministe de haute qualité
Streaming :  l'analyse manuscrit utilise SSE (Server-Sent Events) pour afficher la progression en temps réel
Contexte injecté :  skillsContext ajoute les techniques du Laboratoire Narratif dans chaque prompt système
Température :  modulée par type — créatif (0.85–0.88) · narratif (0.82) · analytique (non défini)
Analyse par type :  paramètre analyseType injecte un focus différent — script doctor (scène) · éditeur littéraire (chapitre) · showrunner (épisode) · comité de lecture (pitch)
H.P.S.A. :  4 axes purs — Humour · Pleur · Suspense · Attractivité — + score global + liste de corrections prioritaires

Notes techniques clés

Codegen Orval

L'index lib/api-zod/src/index.ts est régénéré à chaque codegen. Pour éviter les conflits TS2308, tout requestBody inline doit être converti en $ref vers un named schema dans openapi.yaml.

Migration DB

drizzle push échoue en mode interactif sur Replit. Utiliser executeSql() via le sandbox code_execution pour les migrations directes (ALTER TABLE, CREATE TABLE).

Logging serveur

Jamais console.log côté API. Utiliser req.log dans les route handlers et le singleton logger hors contexte de requête (Pino).

React Query hooks

Les hooks générés par Orval nécessitent un queryKey explicite dans les options query: {}. Utiliser les helpers getGet…QueryKey(id) fournis.

Routing proxy Replit

Tout le trafic passe par le proxy Replit (port 80 → 8080 pour l'API). Ne jamais appeler les ports directs. Utiliser les URLs relatives dans le code frontend.

VPS / Production

Changer gpt-5.4 → gpt-4o dans generationService.ts. Le modèle gpt-5.4 est Replit-only. Utiliser docker compose up -d --build avec .env configuré.
